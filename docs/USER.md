# OnDeck Reps — User Guide

## Who it's for

Parents and guardians of youth baseball players who want to find or offer live reps — pitchers, hitters, and catchers — for at-bats, bullpens, and small group sessions.

## Getting started

1. Visit the home page and **Sign up** with email and password.
2. After signup, you land on your **Dashboard**.
3. Complete your **parent profile** (name, phone, city, state) so other families can reach you.

> **Safety:** Parents/guardians are responsible for supervision, location safety, protective equipment, and session appropriateness. Youth players should never attend sessions without a parent/guardian.

## Player profiles

Add one or more youth players under your account. A player profile is required before posting availability or responding to requests.

### Add a player

Dashboard → **Add Player**, or use the nav → **Add Player**

### Fields you can set

- Name, birth year, age division (9U–18U)
- Roles: pitcher, hitter, catcher (one or more)
- Throws / bats, team level (rec, AA, high school, etc.)
- City, state, travel radius
- Bio

### Edit a player

Dashboard → **Edit** on a player card

## Posting a rep request

Use this when you **need** a pitcher, hitter, catcher, or group session.

### Create a request

Dashboard → **Create Request**, or **Browse Requests** → **Create Request**

### What you specify

- Request type (need pitcher / hitter / catcher / group session)
- Title and description
- Age division and team level
- Date, start/end time
- Location mode: you provide location, provider suggests, or decide together
- Location name, address, city, state
- Spots needed
- Pay type: paid, free, trade reps, or decide together (amount if paid)
- Contact preference: in-app, phone, or email
- Optional safety notes

After posting, the request appears in **Browse Requests** while status is **open**.

### Manage your request

On the request detail page (if you own it):

- View interested responses (message, suggested location/price, contact phone)
- **Mark as filled** when you've found someone
- **Cancel request** if no longer needed

## Browsing & responding to requests

### Browse

Nav → **Requests** (no login required to browse)

### Filter by

- Age division
- Role needed
- Team level
- State
- City (partial match)
- Session date

### Respond to a request

1. Open a request detail page.
2. Log in if needed.
3. Fill out **I'm interested** — pick your player, add a message, optional location/price.
4. The request owner sees your response on their request page.

You need at least one player profile to respond.

---

## Posting availability

Use this when your player **is available** for reps.

### Create availability

Dashboard → **Post Availability**

### What you specify

- Which player
- Available role (pitcher, hitter, catcher)
- Date and time
- City and state
- Whether you can travel (and radius)
- Whether you have field access (with details)
- Pay expectation (paid, free, trade, decide together)
- Notes

Requires at least one player profile. After posting, it appears in **Browse Availability**.

---

## Browsing availability

### Browse

Nav → **Availability**

### Filter by

- Age division
- Role
- State
- Date

Open a post to see full details (player, schedule, travel, location access, pay expectation, notes). There is no in-app "respond" flow for availability yet — families coordinate outside the app using contact info from profiles.

## What you cannot do yet

- Pay through the app
- Background checks
- Map-based search
- Real-time in-app chat
- Push notifications
- Ratings and reviews
- Calendar sync

Families coordinate sessions directly using contact info shown on request responses and profiles.
